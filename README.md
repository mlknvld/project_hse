Анализ предпочтений клиентов при поиске жилья

Данный проект представляет собой исследование данных опроса клиентов, ищущих жилье, с целью:


Выявления ключевых "болей" и предпочтений целевой аудитории.
Построения профилей клиентов на основе их ответов.
Обнаружения логических противоречий в выборе.
Разработки моделей машинного обучения для прогнозирования бюджета и категоризации клиентов.
Выбора оптимальных границ для сегментации бюджета (репрайсинг).
Проект выполнен в среде Jupyter Notebook и содержит полный цикл анализа данных – от предобработки до интерпретации результатов.

Данные

Исходный датасет (course.csv) содержит 1362 записи с ответами респондентов по следующим признакам:

- jtbd_reason_text - Текстовая причина поиска жилья (Jobs-to-be-Done)
- main_pain -	Основная проблема/боль клиента
- max_budget - Максимальный бюджет (в рублях)
- max_commute_time - Максимально допустимое время в пути до офиса (мин)
- metro_priority - Важность близости к метро (от 2 до 5, где 5 – критично)
- preferred_district - Предпочитаемый округ Москвы
- min_rooms -	Минимальное количество комнат
- min_area - Минимальная площадь (кв. м)
- preferred_floor -	Предпочтения по этажу
- property_type - Предпочтения по типу жилья (новостройка/вторичка)
- need_parking - Необходимость парковки
- household_size - Количество проживающих (чел)
- age_group -	Возрастная группа
- current_housing -	Текущая жилищная ситуация
- marital_status - Семейное положение

Методы анализа

- Предобработка и EDA – очистка данных, заполнение пропусков, визуализация распределений признаков.
- Построение профилей клиентов – агрегация по группам «боли», вычисление средних метрик и типичных характеристик для каждого сегмента.
- Выявление противоречий – проверка логических несоответствий в ответах (например, высокая важность метро, но проблема не в дороге).
- Машинное обучение:


  KNN-регрессия для прогноза бюджета (сравнение с весами и без).
  Классификация бюджета с помощью Random Forest, XGBoost, LightGBM, CatBoost.
  Подбор оптимальных границ для категорий бюджета (5 вариантов, включая логарифмическую шкалу).
